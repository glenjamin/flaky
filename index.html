Flaky

A practical introduction to error handling and juggling async

You'll need a couple of dependencies to work on this, request and async.

Be sure to read the docs for both of these for reference.

npm install --save request async
  1. make an HTTP GET request to /list
  2. You will get a 401 with an X-Auth-Token header and a Location: /login
  3. Make an HTTP POST request passing back the X-Auth-Token header to /login with JSON of password = "supersecret"
  4. You'll either get a 200 if correct, or a 401 if wrong
  5. This will flag the token as approved, and you'll now be able to make requests without getting a 401 by including the token header
  6. Make an authenticated request to /list
  7. You'll get a json list of paths back ["/page/1", "/page/2"] etc
  8. Make a request to each of the pages in the response, and calculate an md5 hash. Here's an example.
  9. Post back to the server on /verify with a json object of the numeric part of the page to the md5 hash. { "1": "d0ce1e395418223e6a8d13815714e7cd" }
  10. You'll either get a 200 or a 400 and a true/false for each ID.
